# Security Policy

## Supported Versions

The latest `main` branch and the most recent release tag are supported for security fixes.

## Reporting a Vulnerability

Please report suspected vulnerabilities privately using GitHub Security Advisories:

1. Go to the repository **Security** tab.
2. Click **Report a vulnerability**.
3. Provide a clear description, impact, and reproduction steps.

If Security Advisories are unavailable, open a private report by contacting the maintainers via the repository owner profile and include:

- affected version/commit
- environment details
- proof-of-concept or reproduction steps
- suggested remediation (if available)

## Response Expectations

- Initial triage acknowledgment: within 5 business days
- Status updates: at least weekly while actively triaging
- Fix and coordinated disclosure timing depends on severity and release cadence

Please do not disclose vulnerabilities publicly until a fix is available and maintainers confirm disclosure timing.
